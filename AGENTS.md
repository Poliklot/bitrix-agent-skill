# Инструкции для агентов

## Язык проекта

- Пиши рабочие ответы, документацию, release notes, описания изменений и пояснения **по-русски**.
- Проект ориентирован на русскоязычную аудиторию Bitrix / 1C-Bitrix, поэтому русский язык — основной для коммуникации и справочного слоя.
- Английский используй только там, где он является частью исходного артефакта: API, namespace, class/function names, команды, stack traces, сообщения инструментов, названия GitHub/Git сущностей, ссылки и точные цитаты внешней документации.
- Не переводи имена файлов, путей, модулей, классов, методов, параметров компонентов и shell-команды.
- Если пользователь явно просит другой язык, следуй прямому запросу пользователя.

## Тон

- Формулируй нейтрально и профессионально: “русскоязычная аудитория”, “Bitrix / 1C-Bitrix контекст”, “локальная терминология”.
- Не используй исключающие или оценочные формулировки про национальность, рынок или пользователей.

## Commit messages

- Для commit message учитывай правила из `.agents/rules/commit-messages.md`: conventional commit type, затем русское описание уже выполненного изменения.
