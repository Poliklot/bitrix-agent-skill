# Бытовые Bitrix-карточки

Загружай после `developer-primitives.md` и `first-answer-pitfalls.md` для коротких вопросов “как в PHP/шаблоне/компоненте сделать X”. Формат: **Запрос → Не делай → Делай → Где проверить → Побочные эффекты**. Для готовых project grep-команд используй `core-grep-cookbook.md`; для формата первого ответа — `answer-contracts.md`.

| Запрос | Не делай | Делай | Где проверить | Побочные эффекты |
|---|---|---|---|---|
| meta title/description/canonical/robots | Не печатай meta руками в каждом PHP-файле. | Проверь `ShowHead()` + `ShowTitle()`, свойства страницы/раздела, SEO-параметры компонента, `SetTitle`, `SetPageProperty`. | `header.php`, `.section.php`, параметры `SET_BROWSER_TITLE`, `SET_META_DESCRIPTION`, `component_epilog.php`. | Кеш, composite, SEO-наследование, H1 vs browser title. |
| CSS/JS/head string | Не echo `<link>`/`<script>` из случайного компонента. | `Asset::getInstance()->addCss/addJs/addString` или файлы шаблона компонента. | `ShowHead`, `ShowBodyScripts`, `template_styles.css`, `script.js`. | Порядок подключения, дедупликация, ajax/composite. |
| редактируемый текст | Не хардкодь контент в PHP. | `IncludeFile`, включаемые области, свойства страницы или инфоблок. | `include/`, `local/templates/*/include`, права на файл. | Права, кеш, SITE_DIR, переводимость. |
| компонент/шаблон | Не правь `www/bitrix/modules/*`. | Найди `IncludeComponent`, правь шаблон в `local/templates/.../components/...`; данные — `result_modifier.php`, page effects — `component_epilog.php`. | Вызов компонента, local template, stock component. | Кеш, `setResultCacheKeys`, ajax, composite. |
| breadcrumbs | Не собирай HTML вручную. | `AddChainItem`, `bitrix:breadcrumb`, параметры `ADD_SECTIONS_CHAIN`/`ADD_ELEMENT_CHAIN`. | breadcrumb component, `component_epilog.php`. | Порядок, дубли, ЧПУ, кеш. |
| current user/rights | Не лезь в `$_SESSION`. | `global $USER; IsAuthorized(); GetID();` или project wrapper. | `$USER`, группы, access helpers. | Composite, права групп, персонализация. |
| request GET/POST | Не используй сырой `$_REQUEST` без фильтрации/CSRF. | `Context::getCurrent()->getRequest()`, `check_bitrix_sessid()`. | ajax handlers, controllers, form handlers. | CSRF, XSS, кеш GET-страниц. |
| current URL/query | Не конкатенируй `REQUEST_URI` руками. | `GetCurPage`, `GetCurDir`, `GetCurPageParam` или URL builder. | SEF, `urlrewrite.php`, параметры компонента. | Дубли URL, canonical, `PAGEN_N`, кеш ключи. |
| module include | Не вызывай API отсутствующего модуля. | `Loader::includeModule('iblock')` с обработкой false. | `www/bitrix/modules/<module>/install/version.php`. | Optional modules: `sale`, `catalog`, `currency`. |
| 404 | Не echo “404” и не редирект на главную по умолчанию. | `CHTTP::SetStatus`, `ERROR_404`, проектный `404.php`. | `/404.php`, rewrite, `urlrewrite.php`, strict component params. | SEO, статус 200, composite/cache. |
| redirect | Не выводи HTML до redirect. | `LocalRedirect($url)` или project redirect service. | form handlers, controllers, `SITE_DIR`. | headers sent, open redirect, 301/302, циклы. |
| image resize | Не ограничивайся HTML width/height. | `CFile::GetPath`, `CFile::ResizeImageGet` или image service. | `PREVIEW_PICTURE`, `DETAIL_PICTURE`, UF file, `upload/resize_cache`. | Права, lazy load, webp, cloud storage. |
| iblock property | Не лезь SQL в property tables. | Проверь `PROPERTY_CODE`, `FIELD_CODE`, `DISPLAY_PROPERTIES`; используй iblock API/ORM. | Component params, `$arResult`, `result_modifier.php`, настройки инфоблока. | Кеш, права, активность, тип свойства, escaping. |
| component cache | Не отключай весь кеш первым делом. | Проверь `CACHE_TYPE`, `CACHE_TIME`, `CACHE_GROUPS`, `StartResultCache`, `AbortResultCache`, `setResultCacheKeys`, tagged cache. | `component.php`, params, cache references. | Персонализация, groups, managed tags, composite. |
| mail/form | Не советуй `mail()` как основной путь. | Почтовые события/шаблоны, `CEvent::Send`, `Main\Mail\Event::send`, `form`/project mail service. | mail events, site ID, webforms, queue/agents. | SITE_ID, обязательные поля, charset, spam, queue. |
| AJAX/component action | Не делай endpoint без `prolog_before`, sessid, прав и JSON policy. | Найди project pattern: `BX.ajax`, `runComponentAction`, D7 controller/action или legacy endpoint; проверяй `check_bitrix_sessid`, auth/rights, JSON. | `BX.ajax`, `runComponentAction`, `Controller`, `ActionFilter`, `JsonResponse`, `ajax.php`, `bitrix_sessid`. | CSRF, permissions, composite/ajax mode, cached params, headers. |
| event handler | Не клади бизнес-логику в случайный `template.php`. | `EventManager`, local module/install step или `local/php_interface` для простого legacy; регистрация идемпотентная, обработчик тонкий. | `local/php_interface/init.php`, `local/modules/*/install`, `EventManager::addEventHandler`, `RegisterModuleDependences`. | Повторы регистрации, порядок, runtime context, тяжелые операции. |
| catalog/sale/currency changes | Не меняй цену/заказ/остаток прямым SQL. | Сначала module check `catalog`, `sale`, `currency`; дальше catalog/sale API и project service. | `www/bitrix/modules/{catalog,sale,currency}/install/version.php`, imports, exchange logs. | Пересчёты, скидки, резервы, склад, order history, payment/shipment, 1С/REST. |
| 1С/CommerceML | Не считай upload XML успешным импортом. | Проверить components `catalog.import.1c`, `catalog.export.1c`, `sale.export.1c` и flow `checkauth → init → file → import`. | admin 1c endpoints, exchange logs, `XML_ID`, `CML2_LINK`, active/site/price/stock visibility. | Дубли, price type/currency, stock/reservation, cache/index, schedule. |
